Monitor de Saúde de Dispositivos (Socket & PJL)

Objetivo do Sistema

Este projeto foi desenvolvido para suprir uma necessidade de monitoramento de infraestrutura de TI de forma leve e automatizada. O script testa a conectividade (ping) de uma lista de dispositivos de rede, registrando logs de falha e sucesso em tempo real com uma interface visual dinâmica.

Inovação com Protocolos de Rede

  • Controle de Hardware (PJL): Implementação de comandos de baixo nível via protocolo Printer Job Language para interagir diretamente com impressoras na rede (ex: solicitar página de configuração).
  • Conexão via Sockets: Uso da biblioteca Socket do Python para comunicação direta em portas específicas (ex: porta 9100), garantindo controle total sobre o tráfego de rede.
  • Interface Streamlit: Uso de st.session_state para manter os dados de monitoramento persistentes enquanto o usuário interage com o painel, sem perder o histórico do "ping".
  • Tratamento de Exceções: Lógica robusta com timeouts configuráveis para evitar que dispositivos offline travem a execução do loop de monitoramento.

Visão Técnica

O foco aqui não foi apenas o "Data Analytics", mas sim a Engenharia de Automação para infraestrutura, provando que o Python pode ser uma ferramenta poderosa para manter a disponibilidade de ativos em nível operacional.

Resultados e Valor de Negócio

  • Disponibilidade de Infraestrutura: Monitoramento proativo em tempo real que reduziu em 50% o downtime de dispositivos críticos através da detecção precoce de falhas.
  • Manutenção Remota: Integração com protocolo PJL que permite ações remotas rápidas, economizando custos de deslocamento técnico.
  • Visibilidade de Rede: Painel Streamlit que transformou logs de sistema em indicadores visuais claros para a gestão de TI.
